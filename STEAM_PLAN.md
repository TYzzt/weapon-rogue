# game-rogue Steam 上架规划

## 项目概述

**游戏名称**: 武器替换者 (Weapon Rogue)
**游戏类型**: Roguelike 武器替换闯关游戏
**核心玩法**: 敌人掉落的武器必定替换当前武器
**平台**: PC (Steam)
**目标受众**: Roguelike 爱好者、休闲游戏玩家

## 一、核心系统需求

### 1. 成就系统
- **解锁条件**:
  - 完成第一次游戏
  - 使用每种类型的武器获胜一次
  - 不使用药水通关
  - 达到特定关卡数 (如第10关、第20关)
  - 连续击败X个敌人无受伤
  - 使用稀有度最高的武器获胜

### 2. 存档系统
- **本地存档**:
  - 游戏进度保存
  - 解锁物品记录
  - 高分榜
  - 设置选项
- **云端同步** (可选):
  - Steam 云存档
  - 跨设备同步

### 3. 音效系统
- **背景音乐**:
  - 战斗主题曲
  - 探索主题曲
  - 胜利/失败音乐
- **音效**:
  - 攻击音效 (根据武器类型不同)
  - 受伤音效
  - 武器获取音效
  - 环境音效 (脚步声、开门声等)

### 4. UI/UX 系统
- **主菜单**:
  - 开始游戏
  - 设置
  - 成就
  - 退出
- **暂停菜单**:
  - 继续
  - 重新开始
  - 主菜单
- **HUD**:
  - 血量条
  - 当前武器信息
  - 关卡进度
  - 物品栏

### 5. 平衡性系统
- **武器平衡**:
  - 多样化的武器属性
  - 稀有度与强度的平衡
  - 特殊效果武器
- **敌人平衡**:
  - 递增的难度曲线
  - 多样化的敌人类型
  - AI行为模式调整

## 二、内容量目标

### 1. 武器系统 (目标: 50+ 种武器)
#### 按稀有度分类:
- **普通 (Common)**: 20 种
  - 木棍、生锈的刀、旧扫帚、破布条等
  - 基础伤害，无特殊效果
- **罕见 (Uncommon)**: 15 种
  - 精良短剑、铁锤、长鞭等
  - 轻微特殊效果或更高基础伤害
- **稀有 (Rare)**: 10 种
  - 骑士长剑、魔法法杖、精灵弓等
  - 明显特殊效果
- **史诗 (Epic)**: 4 种
  - 圣剑、龙骨法杖、暗影匕首等
  - 强大特殊效果
- **传说 (Legendary)**: 1 种
  - 混沌之刃、世界之树等
  - 极强能力

#### 按类型分类:
- **近战武器**: 剑、斧、锤、匕首、鞭子
- **远程武器**: 弓、投掷武器、魔法
- **特殊武器**: 诅咒类、辅助类

### 2. 敌人系统 (目标: 20+ 种敌人)
#### 按类型分类:
- **普通敌人**: 史莱姆、哥布林、骷髅兵 (10种)
- **精英敌人**: 史莱姆王、哥布林勇士、骷髅骑士 (6种)
- **Boss**: 关底首领 (4种+)

#### 特征设计:
- 不同攻击模式
- 各自的AI行为
- 掉落特定类型武器

### 3. 关卡系统 (目标: 15+ 个关卡)
- **关卡结构**:
  - 第1-5关: 新手教程区域 (史莱姆森林)
  - 第6-10关: 中级挑战区域 (古老地城)
  - 第11-15关: 高级挑战区域 (恶魔领域)
  - 15关后: 无限模式
- **关卡特色**:
  - 每关都有不同的环境主题
  - 独特的敌人组合
  - 隐藏房间和奖励

### 4. 特殊道具系统
- **消耗品**:
  - 生命药水 (小/中/大)
  - 特殊效果药水 (暂时提升属性)
- **永久升级**:
  - 生命值上限提升
  - 移动速度提升
  - 武器伤害加成

## 三、技术实现方案

### 1. 架构设计
```
src/
├── core/              # 核心游戏逻辑
│   ├── game.js        # 主游戏循环
│   ├── player.js      # 玩家管理
│   └── weapon.js      # 武器系统
├── systems/           # 游戏系统
│   ├── combat.js      # 战斗系统
│   ├── progression.js # 进度系统
│   ├── achievements.js # 成就系统
│   └── save.js        # 存档系统
├── entities/          # 游戏实体
│   ├── player.js      # 玩家实体
│   ├── enemy.js       # 敌人实体
│   └── weapon.js      # 武器实体
├── ui/                # 用户界面
│   ├── hud.js         # HUD系统
│   └── menus.js       # 菜单系统
└── assets/            # 资源文件
    ├── sprites/       # 精灵图片
    ├── sounds/        # 音频文件
    └── data/          # 数据配置文件
```

### 2. 核心功能实现
- **武器替换系统**:
```javascript
// 在玩家接触武器时立即替换
function replacePlayerWeapon(newWeapon) {
  player.currentWeapon = newWeapon;
  // 触发UI更新
  updateWeaponDisplay(newWeapon);
  // 播放音效
  playSound('weapon_pickup');
}
```

- **战斗系统**:
```javascript
function combatSystem(player, enemy) {
  // 玩家攻击敌人
  enemy.takeDamage(player.currentWeapon.damage);
  // 敌人反击
  if (enemy.canAttack()) {
    player.takeDamage(enemy.attackPower);
  }
}
```

### 3. 数据管理系统
- 使用JSON文件管理静态数据
- 动态数据使用对象存储
- 配置分离，便于调整平衡性

### 4. Steamworks 集成
- **成就解锁**: 通过Steam API触发
- **排行榜**: 提交分数至Steam
- **云存档**: 同步存档文件
- **统计信息**: 记录游戏时间、胜利次数等

## 五、质量保证
